**Database Features:**
Complete Library Management: Tracks books, members, borrowing, reservations, and fines

Business Rules Enforcement: CHECK constraints ensure data validity

Referential Integrity: Foreign keys maintain relationship consistency

Scalable Design: Supports multiple copies, authors per book, and hierarchical staff

Real-world Use Case: Models actual library operations with overdue fines, reservations, etc.

**How to Use:**
Save the SQL code in a file named library_management.sql

Execute in MySQL: mysql -u username -p < library_management.sql

Database will be created with all tables and constraints
